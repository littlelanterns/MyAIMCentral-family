# MyAIM Central — Family Platform v2

## Quick Start (Step-by-Step)

### Step 1: Unzip and Open
1. Unzip `MyAIMCentral-family.zip` to your preferred location
2. Open the `MyAIMCentral-family` folder in VS Code

### Step 2: Set Up Credentials
1. Copy `.env.template` to `.env.local`:
   ```bash
   cp .env.template .env.local
   ```
2. Fill in your actual credentials in `.env.local` (see the template for what goes where)
3. The StewardShip credentials are needed for the audit (reading extraction data)
4. The MyAIM v2 credentials get filled in when you create the new Supabase project

### Step 3: Start Claude Code
1. Open the terminal in VS Code
2. Start Claude Code: `claude` (or however you normally launch it)
3. Claude Code will automatically read `CLAUDE.md` at startup (this file gets generated during the audit)

### Step 4: Run the Audit
1. Copy the contents of `workflows/audit-prompt.md` and paste as your first message to Claude Code
2. Stay present for Pass 1 (inconsistency questions) — approximately 1-2 hours
3. For Pass 2 (generation), set up overnight permissions and let Opus work:
   - The `.claude/settings.json` file already has wildcard permissions configured
   - Tell Opus: "Generate all output files. Write questions you can't resolve to questions-for-tenise.md"
4. Wake up to generated files in `/claude`, `/build-prompts`, `/asset-requirements`, and `/workflows`

### Step 5: Review and Answer Questions
1. Open `questions-for-tenise.md` and answer any flagged questions
2. Review the generated `CLAUDE.md` and `claude/` files
3. Review the build prompt order in `build-prompts/`

### Step 6: Build
1. Hand Opus the first build prompt
2. Opus builds, tests, moves to next
3. Come back to this project's Claude chat for strategy discussions
