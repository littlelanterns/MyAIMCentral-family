## Pre-build summary

**Headline:** Next Best Thing (PRD-25's unhideable flagship feature) has been fully dark for every Guided kid since 2026-05-03 because `useNBTEngine.ts` never applied the day-scheduling filter (`filterTasksForToday`) that `GuidedActiveTasksSection` already uses — so an MWF routine could get suggested on a Tuesday. Root-cause fix + re-enable is Slice 1. LiLa Homework Help, Communication Coach, header gamification indicators, and task point values are ALREADY wired (recon found the STUB_REGISTRY entries were stale) — Slice 4 corrects the registry. Messages tab redirect-to-`/messages` is ratified as the PERMANENT design (not an interim placeholder). Progress bottom-nav tab is retargeted from the `GuidedProgress.tsx` warm stub to the real `/my-rewards` page (KIDS-REWARDS-PAGE build) — one rewards home per kid. Victories is restored to the primary 5-tab bottom nav per Convention #124 (already documented, code had drifted to 4 tabs + Victories in the More menu). DailyCelebration's Steps 2.5 (Reflections), 3 (Streak), 4 (Theme/sticker-book progress) are un-stubbed together since the underlying data (compute_streak, member_sticker_book_state, reflection prompts) is all real now.

**Decisions (all RESOLVED per pack recommendations):**
- D-GDCX-1: Messages tab keeps the redirect-to-`/messages` pattern, permanently.
- D-GDCX-2: Progress nav tab → `/my-rewards`; `GuidedProgress.tsx` retires.
- D-GDCX-3: Victories restored to the primary Guided bottom nav (Convention #124 already says this — code catches up).
- D-GDCX-4: DailyCelebration Steps 2.5 + 3 + 4 unstub together, same file, same pass.
- D-GDCX-5: Per-prompt reflection toggles stay an accepted simplification (the existing "N prompts per day" count dropdown stands); the dead `reflection_prompts`/`reflection_custom_prompts` preference array fields are noted for future cleanup, not touched this build.

**One schema decision made during implementation (not in the original pack, which stated "Migrations: None"):** `reflection_responses.source_context` has a DB-level `CHECK (source_context IN ('reflections_page', 'evening_rhythm'))` constraint (migration `00000000100071`). PRD-25 Screen 6 explicitly specifies celebration-originated reflections should carry a distinguishing tag (`journal_entries.source = 'daily_celebration'`). `journal_entries.source` itself is a free TEXT column (no CHECK), but `reflection_responses.source_context` needs its enum extended to accept `'daily_celebration'` as a legal value, or the PRD's explicit tagging language can't be honored. Added one tiny idempotent migration extending the CHECK constraint (additive only, no data migration, no destructive change) rather than mis-tagging celebration-originated responses as `'reflections_page'`.
