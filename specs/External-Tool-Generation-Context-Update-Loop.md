# External Tool Generation & Context Update Loop

**Created:** March 13, 2026
**Related PRDs:** PRD-05C (LiLa Optimizer), PRD-19 (Family Context & Relationships)
**Purpose:** Documents the pattern for generating AI tools on external platforms (Claude Projects, Gemini Gems, ChatGPT Custom GPTs) that include built-in context update instructions, creating a natural retention loop.

---

## Overview

MyAIM's LiLa Optimizer doesn't just craft one-off prompts — it can generate complete, deployable AI tools that users set up on their own AI platform accounts. These tools run at the user's own expense (their Claude Pro/ChatGPT Plus/Gemini Advanced subscription), but the intelligence that powers them — the family context, the framework design, the communication guides — comes from MyAIM.

The key innovation is the **context update loop**: every generated tool includes instructions for receiving context updates from MyAIM, and MyAIM's monthly freshness review generates pre-formatted update blocks for all deployed tools. This means the tool degrades over time without an active MyAIM subscription, creating a natural retention hook that's genuinely valuable (not manipulative — the updates are real improvements based on real family changes).

---

## How It Works

### Step 1: User Selects a Tool to Deploy

From the AI Toolbox (or via the Optimizer), mom selects a tool she wants to create on an external platform. Examples:

- **Board of Directors** — a thinking tool where she defines advisors (real or archetypal) to weigh in on decisions
- **Six Thinking Hats** — structured multi-perspective analysis
- **Decision Matrix** — weighted decision-making framework
- **Homework Helper** — teen-facing tool that teaches without giving answers
- **Meal Planner** — family-context-aware meal planning
- **ThoughtSift** — emotional/decision processing partner

### Step 2: User Chooses Target Platform

Mom selects where she wants to deploy:

- **Claude** — generates a Project Instructions block to paste into a new Claude Project
- **ChatGPT** — generates Custom GPT instructions (or Custom Instructions for simpler tools)
- **Gemini** — generates Gem configuration instructions

Each platform has different formatting requirements. The Optimizer handles platform-specific formatting automatically.

### Step 3: Optimizer Assembles the Tool

The Optimizer generates a complete tool package containing three sections:

**Section A: Framework Instructions**
The tool's behavioral rules, conversation structure, and personality. This is the "what the tool does" part. Example for Board of Directors:

```
You are facilitating a Board of Directors session. The user has assembled 
a virtual advisory board to think through a decision or challenge.

Board members:
- [Name/Role 1]: [Perspective description]
- [Name/Role 2]: [Perspective description]
- [Name/Role 3]: [Perspective description]

For each board member, adopt their perspective and provide advice that 
reflects their expertise, values, and thinking style. After all board 
members have weighed in, synthesize the key insights and help the user 
identify their own decision.

Always end with: "What resonates with you from these perspectives? 
What's your instinct telling you?"
```

**Section B: Family Context Block**
The user's current family context, assembled from Archives, formatted for the target platform. This includes relevant context items, "How to Reach Me" cards for involved family members, relationship notes (mom's own), and any other context the Optimizer determines is relevant for this specific tool.

All real names are replaced with external aliases where `use_alias_for_external = true`.

The context block is clearly delimited with markers:

```
=== FAMILY CONTEXT (Generated by MyAIM on 2026-03-13) ===

Family: The Wertman Family (9 children, homeschooling)

About Student A (age 14, Independent):
- Visual learner, loves Minecraft
- ENFP — enthusiastic but scattered
- Love language: Quality Time
- Responds best to humor and directness
- Currently excelling in math
- Allergic to peanuts (EpiPen in backpack)

About Student B (age 12, Guided):
- Kinesthetic learner
- Quiet, needs processing time before responding
- Motivated by mastery, not competition
...

=== END FAMILY CONTEXT ===
```

**Section C: Context Update Instructions**
Instructions embedded in the tool itself that tell the AI platform how context updates work:

```
=== CONTEXT UPDATE INSTRUCTIONS ===

This tool's family context was generated by MyAIM Family on [date].
Family context changes over time — kids grow, preferences shift, 
schedules change.

TO UPDATE THIS TOOL'S CONTEXT:
1. Open MyAIM Family → AI Toolbox → Deployed Tools
2. Find this tool and tap "Generate Context Update"
3. Review the suggested changes (additions, revisions, removals)
4. Approve the updates
5. Copy the new context block and replace the FAMILY CONTEXT section 
   above (between the === markers)

Your next context update will be available after your [next billing date] 
monthly review.

Note: This tool works best with current context. If the family context 
is more than 2-3 months old, some recommendations may not reflect 
recent changes in your family.

=== END CONTEXT UPDATE INSTRUCTIONS ===
```

### Step 4: User Deploys the Tool

The Optimizer provides platform-specific setup instructions:

**For Claude:**
1. Go to claude.ai → Projects → New Project
2. Name it "[Tool Name]"
3. Paste everything above into the Project Instructions
4. Start chatting in the project — Claude will follow the framework with your family context
5. Copy the project URL from your browser and paste it back in MyAIM so you can launch it from your toolbox anytime

**For ChatGPT:**
1. Go to chat.openai.com → Explore GPTs → Create
2. Name it "[Tool Name]"
3. Paste the Framework Instructions into the Instructions field
4. Paste the Family Context and Update Instructions into the Knowledge section
5. Save and start using
6. Copy the GPT URL and paste it back in MyAIM so you can launch it from your toolbox anytime

**For Gemini:**
1. Go to gemini.google.com → Gems → New Gem
2. Name it "[Tool Name]"
3. Paste everything above into the Gem instructions
4. Save and start using
5. Copy the Gem URL and paste it back in MyAIM so you can launch it from your toolbox anytime

### Step 5: Monthly Context Update Loop

On each billing day, when the monthly context freshness review runs:

1. **Aggregation compiles the month's data** (victories, task completions, educational hours, etc.)
2. **Freshness review compares** aggregated data against each member's current context
3. **For each deployed external tool**, the system generates a **pre-formatted context update block** that:
   - Shows what changed since the last update (additions, revisions, removals)
   - Provides a complete replacement FAMILY CONTEXT section ready to paste
   - Is formatted for the specific platform the tool is deployed on

4. **Mom receives a notification:** "3 of your external tools have context updates available."
5. **Mom reviews** the updates in-app:
   - Per-tool update cards showing the diff ("Student A: 'struggles with math' → 'excelling in math'")
   - One-tap approve per tool
   - Copy updated context block to clipboard
   - Platform-specific instructions for where to paste

6. **Mom pastes** the updated context block into her external tool, replacing the old FAMILY CONTEXT section.

## Data Model

### External Tool Registry

The Optimizer needs to track which tools the user has deployed externally.

**Stored in `optimizer_external_tools` table (or as JSONB on an existing Optimizer table):**

| Field | Type | Notes |
|-------|------|-------|
| id | UUID | PK |
| family_id | UUID | FK → families |
| created_by | UUID | FK → family_members |
| tool_name | TEXT | User-chosen name for this deployment |
| tool_type | TEXT | Which framework (board_of_directors, six_hats, decision_matrix, etc.) |
| target_platform | TEXT | 'claude_project', 'chatgpt_custom_gpt', 'gemini_gem', 'other' |
| tool_url | TEXT | URL to the deployed tool on the external platform. Mom pastes this after setup. Makes the tool launchable directly from MyAIM. |
| context_snapshot | JSONB | The family context that was included at generation time |
| context_generated_at | TIMESTAMPTZ | When the context was last generated |
| alias_mapping | JSONB | Which aliases were used (for update consistency) |
| member_ids_included | UUID[] | Which family members' context is in this tool |
| is_active | BOOLEAN | Whether this tool is still being maintained |
| created_at | TIMESTAMPTZ | |
| updated_at | TIMESTAMPTZ | |

### Tool Bookmarking & Launch

After deploying a tool, the setup instructions include a final step:

```
6. Copy the URL of your new [Project/GPT/Gem] and paste it below 
   so you can launch it directly from MyAIM anytime.

   [Paste tool URL here: ________________________________]
```

Mom pastes the URL (e.g., `https://claude.ai/project/abc123` or `https://chatgpt.com/g/xyz789`). This is stored in `tool_url` on the external tool registry.

**In the AI Toolbox, deployed external tools appear as launchable cards:**

```
┌─────────────────────────────────────────────┐
│  🚀 My Deployed Tools                       │
│  ─────────────────────────────────────────  │
│  📋 Board of Directors          Claude ↗    │
│     Context updated: Mar 1, 2026            │
│     ⚠️ Update available                     │
│  ─────────────────────────────────────────  │
│  🎓 Homework Helper             Gemini ↗    │
│     Context updated: Mar 1, 2026            │
│     ✓ Up to date                            │
│  ─────────────────────────────────────────  │
│  🍽️ Family Meal Planner        ChatGPT ↗   │
│     Context updated: Feb 1, 2026            │
│     ⚠️ Update available (2 months old)      │
│  ─────────────────────────────────────────  │
└─────────────────────────────────────────────┘
```

Each card shows:
- Tool name and icon
- Platform badge with external link arrow (↗) — tapping opens the tool URL in a new tab
- Context freshness indicator (date of last update)
- Update status: "✓ Up to date" or "⚠️ Update available" with age indicator
- Tapping the update badge opens the context update review flow
- Long-press or menu: edit tool name, update URL, deactivate, delete

**MyAIM becomes the bookmark hub for all of mom's AI tools** — both internal (LiLa-powered) and external (deployed on other platforms). One place to launch everything, one place to manage context updates.

> **Mom experience goal:** I don't want to hunt through my Claude projects or my ChatGPT sidebar to find my Board of Directors tool. I open MyAIM, tap the card, and I'm there. And when the context is stale, MyAIM tells me and gives me the update in one tap.

This table enables the monthly review to know:
- Which tools exist and what platform they're on
- Which members' context they contain (so only relevant changes are flagged)
- What the last context snapshot looked like (so we can generate a meaningful diff)
- Which aliases were used (so updates use the same aliases)

## Retention Dynamics

### Why This Creates Healthy Retention

The context update loop creates retention through **genuine value**, not manipulation:

1. **Context degrades naturally.** Kids change. Schedules shift. A 3-month-old context snapshot about a 14-year-old is meaningfully outdated. The updates are real improvements.

2. **The tool framework is yours to keep.** If someone cancels, they keep the tool with its last context snapshot. It still works — just with stale context. No data destruction, no feature lockout.

3. **Fresh context is the subscription value.** The monthly update is the tangible deliverable that justifies the subscription. "Your October context update is ready — Jake's math confidence is up, Emma started dance, and the family schedule shifted. Update your 3 external tools in one tap."

4. **Compound value over time.** The longer someone subscribes, the more external tools they've deployed, the more context updates they rely on. Switching costs increase naturally through genuine utility.

### What Happens on Cancellation

- All deployed tools continue to work with their last context snapshot
- No context updates are generated after cancellation
- The tools degrade gracefully — they're still useful, just increasingly outdated
- If the user resubscribes, context updates resume from the current state (not from where they left off — the system regenerates a full current context block)
- Win-back messaging can reference specific tools: "Your Board of Directors tool is running on 4-month-old context. Student A has had a big quarter — want to update?"

## Future Enhancements

- **Auto-paste via API:** If platforms eventually support API-based instruction updates, MyAIM could push context updates directly without mom needing to copy-paste. This is platform-dependent and not available today.
- **Tool performance tracking:** If the external platform supports usage analytics (or if mom self-reports), track which tools she uses most and surface them prominently.
- **Tool sharing between families:** A mom could share her "Board of Directors" framework (without family context) with other MyAIM users. The framework is reusable; the context is personal.
- **Tool marketplace in AI Vault:** Power users create tool frameworks that other families can browse and deploy with their own context. This ties into the AI Vault's content discovery model.
- **Context update via QR code or deep link:** Instead of copy-paste, generate a QR code or link that opens the external platform and pre-fills the update. Platform-dependent.

## Key Conventions

- External tool context blocks are always delimited with `=== FAMILY CONTEXT ===` markers for easy replacement
- Context update instructions are always included in every generated tool — they're not optional
- Aliases are consistent across updates (same member always gets the same alias in the same tool)
- The monthly freshness review generates update blocks ONLY for active tools (`is_active = true`)
- Platform-specific formatting is handled by the Optimizer, not by the user
- No family context is ever sent to MyAIM's servers from the external platform — the data flow is one-directional (MyAIM → external platform, never the reverse)
